Problem Statement

The purpose of this project is to create a template for the T-Mobile internal team "EIA" to make their own website/portal to house all of their projects and information.

Goals

Responsive

The clients for this template requested responsiveness, so it should not only be smooth in its transitions, but also seamlessly transition to a mobile version as needed.

Simple

This webpage is meant for viewing by a variety of people, so it should not be too complex to navigate.

Easy to Edit

Since the requested product is only a template, it should be designed in such a way as to be fairly easy for someone else to pick up and make edits and additions.

Expandable

The scope of this website is unclear, so it should be programmed in such a way so as to not box in its design and be easily expanded should more pages be needed in the future.

Initial Implementation

Initially, I created a webpage from scratch that would serve as a template for the EIA team to edit and maintain.
Click images to view full versions in a new tab.
It became clear as production continued, however, that the webpage was likely going to be run by somebody with little to no programming experience, which led to a new design challenge: how can I create a template website that is user agnostic? Having a pure HTML solution was no longer feasible, so the design needed to be completely changed to reflect that.

Solutions

Confluence

The solution I came upon to solve the usability issue of this template was to reconstruct a simplified version of it in Confluence, T-Mobile's local page creation tool. Confluence is normally meant only for Wiki-style pages, but with a bit of creative CSS and JavaScript usage, I was able to work around that limitation and turn it into a rudimentary website editor. The limitations of the tool meant some design choices had to be simplified a bit in order to be maintained in the new version, but otherwise meant that even a non-tech savvy user would be able to edit the webpage without much issue.
